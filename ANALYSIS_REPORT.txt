BROKER HOLDINGS DATA - ANALYSIS & FIX REPORT
=============================================

ANALYSIS DATE: 2026-06-26
ISSUE: Broker holdings table (stock ACLBSL) shows incorrect data

===============================
FINDINGS SUMMARY
===============================

1. MISSING SELL DATA (CRITICAL)
   Root Cause: Data not synced from floorsheet_trades table
   Impact: Cannot calculate net position accurately
   Severity: CRITICAL (affects all broker analysis)
   Evidence: Many records show buyQty > 0 but sellQty = 0

2. MISSING TRANSACTION COUNTS (HIGH)
   Root Cause: Database missing buyContracts/sellContracts columns
   Impact: Cannot identify trade concentration
   Severity: HIGH (affects trade pattern analysis)

3. MISSING MATCHING VOLUME (MEDIUM)
   Root Cause: Not calculated in aggregation logic
   Impact: Cannot identify internal/matched trading
   Severity: MEDIUM (useful for market analysis)

4. TIME RANGE AGGREGATION USES WRONG FORMULA (HIGH)
   Root Cause: API handler uses SUM for prices instead of VWAP
   Impact: Average prices incorrect for multi-day ranges
   Severity: HIGH (affects trend analysis)

5. CUMULATIVE NET CALCULATION INCONSISTENT (MEDIUM)
   Root Cause: Query assumes continuous daily records
   Impact: Cannot track historical holding buildup
   Severity: MEDIUM (breaks historical analysis)

===============================
DATA FLOW ANALYSIS
===============================

Source Data:
  floorsheet_trades (buyer, seller, qty, amount)
           |
         [sync job] - INCOMPLETE
           |
  broker_daily_agg (date, symbol, broker, buyQty, sellQty, etc.)
           |
  /api/broker-stocks endpoint - WRONG AGGREGATION
           |
  UI: holding/page.tsx - MISSING DATA

Issue Points:
  1. floorsheet_trades → broker_daily_agg
     - Sell quantity missing for some brokers
     - No transaction counts captured

  2. broker_daily_agg → API response
     - Uses SUM for all metrics (wrong for averages)
     - No contract counts returned
     - No matching volume calculation

  3. API → UI
     - UI awaiting data that doesn't exist
