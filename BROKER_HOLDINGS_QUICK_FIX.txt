BROKER HOLDINGS DATA - QUICK REFERENCE FIX GUIDE
================================================

PROBLEM SUMMARY
---------------
Broker holdings table (stock ACLBSL) shows incorrect data:
- Sell Vol/Sell Amt always 0 for all brokers (CRITICAL)
- No transaction counts (Buy/Sell contracts)
- No matching volume/ratio calculations
- Time-range aggregation uses wrong calculation method (not VWAP)
- Cumulative net holdings incorrect

DATA ISSUES - DETAILED
----------------------

Issue 1: Missing Sell Data (ALL zeros)
  Root: Incomplete sync from floorsheet_trades
  Fix: Backfill from floorsheet_trades table
  SQL: UPDATE broker_daily_agg SET sellQty = (SELECT SUM(...))

Issue 2: No Transaction Counts
  Root: Database missing buyContracts/sellContracts columns
  Fix: ALTER TABLE + backfill from floorsheet_trades
  Impact: Cannot see trade concentration (1 big order vs 100 small)

Issue 3: No Matching Volume
  Root: Not calculated in aggregation
  Fix: Calculate MIN(buyQty, sellQty) per record
  Impact: Cannot identify internal/matched trades

Issue 4: Wrong Time Range Aggregation
  Root: Uses SUM for prices (should use VWAP)
  Problem:
    Day 1: 100 shares @ 50 = 5000
    Day 2: 100 shares @ 60 = 6000
    Wrong: 50 + 60 / 2 = 55 ✗ (if using daily avgs)
    Right: 11000 / 200 = 55 ✓ (VWAP)
  Fix: Use SUM(amount) / SUM(qty) for ranges

Issue 5: Cumulative Net Wrong
  Root: Assumes continuous daily records (they're not)
  Problem: If broker doesn't trade on Day 2, cumulative breaks
  Fix: Ensure all dates 1-to-target in query, fill zeros


SQL QUERIES FOR VERIFICATION
-----------------------------

Check for missing sells:
  SELECT brokerId, COUNT(*) FROM broker_daily_agg
  WHERE sellQty = 0 AND buyQty > 0
  GROUP BY brokerId;

Verify floorsheet consistency:
  SELECT bda.brokerId, bda.buyQty as agg, fs.actual_qty
  FROM broker_daily_agg bda
  LEFT JOIN (
    SELECT buyerMemberId, SUM(contractQuantity) as actual_qty
    FROM floorsheet_trades
    WHERE tradeDate = '2026-06-26' AND stockSymbol = 'ACLBSL'
    GROUP BY buyerMemberId
  ) fs ON bda.brokerId = fs.buyerMemberId
  WHERE bda.tradeDate = '2026-06-26' AND bda.stockSymbol = 'ACLBSL';

Test VWAP calculation:
  SELECT
    brokerId,
    SUM(buyQty) as totalQty,
    SUM(buyAmt) as totalAmt,
    ROUND(SUM(buyAmt) * 1.0 / SUM(buyQty), 2) as vwapPrice
  FROM broker_daily_agg
  WHERE tradeDate >= '2026-06-24' AND tradeDate <= '2026-06-26'
  GROUP BY brokerId;

TROUBLESHOOTING
---------------

Q: Fixer still shows zero sells for some brokers after running?
A: Some brokers may have one-sided trades (only buy OR only sell).
   This is legitimate. Check with SQL query to verify against floorsheet_trades.

Q: Performance degraded after running fixer?
A: Add index on (tradeDate, brokerId, stockSymbol) if missing
   > CREATE INDEX idx_bda_lookup ON broker_daily_agg(tradeDate, brokerId, stockSymbol);

Q: Cumulative net still looks wrong?
A: Run this verification query:
   > SELECT MIN(tradeDate) FROM broker_daily_agg WHERE stockSymbol = 'ACLBSL';
   Then manually sum netQty from that date to target date.

Q: API returns null/missing fields?
A: Ensure all columns were added and backfilled:
   > SELECT * FROM broker_daily_agg LIMIT 1;
   Should show: buyContracts, sellContracts, matchingVolume, matchingAmt
